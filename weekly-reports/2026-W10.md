### 关键设计决策

**FE约束重新理解：**
```python
# 正确约束：finish_cycle必须严格大于上一事务的finish_cycle
finish_cycle = input_cycle + latency + 1
约束：new_finish > last_finish  # 不是 >=，是 >
```

**案例分析：**
| Cycle 1 lat | Cycle 2 lat | finish_cycle | 结果 |
|-------------|-------------|--------------|------|
| 2 | 1 | 4 vs 4 | ❌ 冲突 |
| 2 | 0 | 4 vs 3 | ✅ 可调度 |
| 2 | 2 | 4 vs 5 | ✅ 可调度 |

### V3.2 详细改进

**多FE完成处理 - 优先级编码器：**
```python
fe_done = u(1, 0)
fe_done_data = u(DATA_WIDTH, 0)
fe_done_seq = u(SEQ_WIDTH, 0)

for i in range(N_FE):  # FE0 > FE1 > FE2 > FE3
    is_done = fwded_vld[i]
    fe_done = fe_done | is_done
    fe_done_data = fe_done_data if (~is_done).out() else fwded_data[i]
    fe_done_seq = fe_done_seq if (~is_done).out() else fe_pkt_seq[i].out()
```

**依赖查找实现：**
```python
# CAM结构查找被依赖报文
target_seq = ic_seq[i].out() - dep
dep_found = u(1, 0)
dep_value = u(DATA_WIDTH, 0)
for j in range(MAX_DEP + 1):
    match = dep_valid[j] & (dep_seq[j] == target_seq)
    dep_found = dep_found | match
    dep_value = dep_value if (~match).out() else dep_data[j]
```

## 三、3月9日（周一）架构重构：V3.4

### 问题识别
原有架构（V3.0-V3.2）将FE端口暴露在顶层，不符合实际使用场景：
- 顶层端口包含 `fwdedN_pkt_data_vld` 等FE反馈信号
- FE应该是内部模块，不应由外部驱动

### 架构改进：FE封装为内部黑盒

**新架构设计：**
```
fastfwd_v3_4 (Top Level Wrapper)
├── 顶层端口（仅lane相关）：
│   ├── Input:  lane[0-3]_pkt_in_vld/data/ctrl
│   ├── Output: lane[0-3]_pkt_out_vld/data
│   └── Output: pkt_in_bkpr
│
├── u_core: fastfwd_v3_4_core (pyCircuit生成)
│   ├── 输入：lane输入、fwded_xxx（来自FE）
│   ├── 输出：lane输出、fwd_xxx（到FE）、bkpr
│   └── 内部：Input Collector, FE Scheduler, Dependency Table, ROB
│
└── u_fe0~u_fe3: 4个FE实例（golden模块）
    └── 连接到core的fwd/fwded信号
```

### 实现细节

**文件结构：**
```
verilog_auto/
├── fastfwd_v3_4.v          # 顶层wrapper，实例化4个FE
├── fastfwd_v3_4_core.v     # pyCircuit生成的控制逻辑
└── fastfwd_v3_4_core.v.stats.json  # 时序统计
```

**端口对比：**

| 端口类型 | V3.2 | V3.4 |
|---------|------|------|
| Lane输入 | ✅ 4 lane | ✅ 4 lane |
| Lane输出 | ✅ 4 lane | ✅ 4 lane |
| FE反馈 | ❌ 暴露在顶层 | ✅ 内部连接 |
| FE控制 | ❌ 暴露在顶层 | ✅ 内部连接 |
| 背压 | ✅ | ✅ |

**复位处理：**
- Core使用 `rst` (active-high)
- FE模块使用 `rst_n` (active-low)
- Wrapper内部转换：`rst_n = ~rst`
